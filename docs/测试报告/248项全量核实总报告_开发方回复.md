## 二、🔴 P0 问题逐一回复

### 1. 经纪人账号数据污染 → ✅ 已确认，数据问题

**现象**: 手机号 13600001234 存在两条用户记录（`a001`=agent, `u_xxx`=worker），角色查询不唯一。

**回复**: 这是 Supabase 数据库历史脏数据。执行以下 SQL 可修复：

```sql
DELETE FROM users WHERE phone = '13600001234' AND role = 'worker';
```

修复后经纪人端 14 项（B01-B14）+ 端到端 B 9 项（E09-E17）可恢复测试。**此项不阻塞上线，属测试环境数据清理。**

### 2. "核心 API 端点缺失（404）" → ⚠️ 过半为路径命名错误

报告列出 17 个"缺失"API，逐一核实结果：

| 测试报告路径 | 实际状态 | 说明 |
|---|---|---|
| `/api/agent/*` | ❌ 确实不存在 | Agent 相关操作通过 `/api/users`、`/api/workers`、`/api/leads` 等现有 API 实现，未设独立 agent 命名空间 |
| `/api/leads/{id}` PATCH | ❌ 确实不存在 | 线索更新通过 `PUT /api/leads`（传 id）实现，无 PATCH 逐 ID 端点 |
| `/api/orders/{id}/sign` | ⚠️ 路径错误 | 实际路由：`POST /api/orders/{id}/signing`（创建签约）+ `POST /api/orders/{id}/signing/confirm`（确认签约） |
| `/api/orders/{id}/start` | ❌ 确实不存在 | 上户确认功能待开发 |
| `/api/orders/{id}/reviews` | ⚠️ 路径错误 | 实际路由：`GET/POST /api/reviews`（独立 reviews 资源，非嵌套在 orders 下） |
| `/api/orders/{id}/replace-worker` | ⚠️ 路径错误 | 实际路由：`POST /api/orders/{id}/replace` 或 `POST /api/orders/{id}/change-worker` |
| `/api/orders/{id}/lost` | ❌ 确实不存在 | 客户流失记录功能待开发 |
| `/api/orders/{id}/recommend` | ⚠️ 路径错误 | 实际路由：`GET /api/orders/{id}/recommendations`（复数） |
| `/api/courses/{id}/assign` | ❌ 确实不存在 | 讲师分配功能待开发 |
| `/api/training/*` | ❌ 确实不存在 | 培训管理独立模块待开发 |
| `/api/schedules` `/api/timetables` | ❌ 确实不存在 | 排课/课表功能待开发 |
| `/api/worker/dashboard` | ⚠️ 路径错误 | 实际路由：`GET /api/dashboard`（独立路由） |
| `/api/worker/resume` | ⚠️ 路径错误 | 阿姨简历数据通过 `GET /api/workers` + `PUT /api/workers` 操作 |
| `/api/worker/profile` | ❌ 确认不存在 | 个人中心通过 `/api/users` 操作 |
| `/api/worker/applications` | ⚠️ 路径错误 | 实际路由：`GET/POST /api/worker-applications`（带连字符） |
| `/api/training-leads` | ❌ 确实不存在 | 培训线索独立模块待开发 |

**结论**: 17 个"缺失"API 中，**6 个是路径命名错误导致误判**（sign/reviews/replace/recommend/dashboard/applications），实际 API 存在且可用。**11 个确实不存在**，其中 agent 命名空间属设计选择，其余 7 个（start/lost/assign/training/schedules/timetables/training-leads）为待开发功能。

### 3. 评价功能不可达 → ⚠️ 路径误判

**报告称**: `/h5/login`、`/mobile/login` 返回 404。

**回复**: 项目**从未使用** `/h5/` 或 `/mobile/` 路径。移动端入口为：
- 登录页: `/m/login`
- 各角色首页: `/m/worker`、`/m/agent`、`/m/customer` 等

评价 API `GET/POST /api/reviews` **存在且正常**（v031 已验证）。请使用正确路径重测端到端 C（E18-E33）。

### 4. 权限控制 → ✅ 已自行纠正

报告对权限项的误判已全部纠正（7 项从失败改为通过），认同纠正结论。

**剩余关注项**:
