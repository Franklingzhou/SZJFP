## 3. 活跃BUG清单

### 🔴 P0（2个，阻塞上线）

#### BUG-46：合同全生命周期API路由缺失（BUG-44未修复）

| 项目 | 内容 |
|------|------|
| 优先级 | P0 |
| 对应第四轮 | BUG-44 |
| 046报告声称 | "权限矩阵042已修" |
| 实际验证 | ❌ 未修复 |
| 现象 | 所有合同状态变更API路由全部返回404 |
| 影响范围 | 合同永远停留在draft状态，无法签署、审批、驳回 |
| 复现步骤 | 1. 创建合同POST /api/contracts → 成功(draft) <br> 2. POST/PATCH/PUT /api/contracts/{id}/approve → 404 <br> 3. POST/PATCH/PUT /api/contracts/{id}/sign → 404 <br> 4. PATCH/PUT /api/contracts/{id} body={status:"signed"} → 404 |
| 试过的方法 | POST/PATCH/PUT × approve/sign/reject/PATCH到合同本体 × 空/body → 全部404 |
| 数据佐证 | 所有合同approved_by=null, signed_at=null，仅1条rejected状态(可能是手动数据库操作) |

#### BUG-47：CRUD API路由大面积缺失（BUG-45未修复）

| 项目 | 内容 |
|------|------|
| 优先级 | P0 |
| 对应第四轮 | BUG-45 |
| 046报告声称 | "leads/courses/orders DELETE 200" |
| 实际验证 | ❌ 未修复，DELETE全部404 |
| 现象 | 所有资源ID级别的PUT/PATCH/DELETE路由返回404(HTML页面) |
| 影响范围 | 无法编辑或删除任何已创建的资源(线索/课程/订单/合同/合同模板/worker等) |
| 复现步骤 | 1. 创建线索 POST /api/leads → 成功 <br> 2. PUT/PATCH /api/leads/{id} → 404 <br> 3. DELETE /api/leads/{id} → 404 <br> 同样适用于courses/orders/contracts/contract-templates/workers |
| OPTIONS验证 | OPTIONS /api/leads → "allow: DELETE, GET, HEAD, OPTIONS, POST, PUT" (声明支持但实际404) |
| 根因推测 | Next.js API路由配置问题：只注册了集合级路由(GET/POST)，未注册资源级路由(PUT/PATCH/DELETE /{id}) |
