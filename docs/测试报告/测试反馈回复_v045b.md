## 三、对测试人员的回答

### ✅ 已确认并修复的（3个真bug）

1. **简历详情报错** → 已修。工种字段从DB出来是字符串`"保姆,月嫂"`，前端误当数组调`.join()`，加了类型兼容。
2. **培训管理 404** → 已修。建了入口页，自动跳转到课程管理。
3. **证书 API 缺失** → 已补建 `/api/certificates` GET 路由。

### 🟡 不是bug，属于理解偏差的（3个）

4. **`/api/auth/login` 404** → 我们系统登录接口叫 `/api/auth/phone-login`，从来没有 `/api/auth/login`。不是bug。
5. **评价页没有课程审核Tab** → 课程审核入口在侧边栏"课程管理"，跟评价管理是两个独立模块，设计就是这样。不是bug。
6. **/admin/review 和 /admin/evaluation 404** → 正确的地址是 `/admin/reviews`（有s）。这些路径从未规划过，不存在是正常的。不是bug。

### 🟡 功能存在但入口不明显（2个）

7. **"标记流失"按钮** → 代码里有，是列表行末的切换图标（ToggleLeft/ToggleRight），hover 提示"标记流失"。确实不够直观，后续版加文字标签。
8. **数据导出按钮** → API `/api/workers/export` 是好的，但前端没放导出按钮。后续补上。

### ⚠️ 待进一步确认（2个）

9. **合同"关闭"按钮无响应** → 从代码看逻辑完整（confirm → API → alert），怀疑是弹窗遮罩层拦截。需在部署后现场重现。
10. **订单签约弹窗遮罩** → 需看具体页面的 z-index 层级，待复现确认。
