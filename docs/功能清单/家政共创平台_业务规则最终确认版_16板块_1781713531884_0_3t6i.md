# 家政共创平台 全业务规则最终确认版（16板块 · 2026-06-17 周总确认）

---

## 一、8个角色定位

| 角色 | 核心定位 |
|------|----------|
| **经纪人（agent）** | 核心中间人，连接客户和阿姨，唯一能发单的角色 |
| **招生（recruiter）** | 独立角色，负责招聘学员和阿姨，线索转学员，推荐阿姨给订单 |
| **讲师（instructor）** | 培训阿姨，管理课程和学员，培训合格后转简历上户 |
| **阿姨运营（worker_operator）** | 管理阿姨档案，线索录入转简历，在推荐大厅推荐阿姨 |
| **培训主管（training_supervisor）** | 审批合同/课程，全量查看线索/学员/课程，也可转简历上户 |
| **阿姨（worker）** | 被服务对象，在订单大厅投递/自荐，管理自己简历 |
| **客户（customer）** | 被服务对象，只看已签约订单，不能进订单大厅 |
| **管理员（admin）** | PC端后台，全平台管理，审核所有操作 |

## 二、招生培训线（线索→学员→阿姨）

| 状态变化 | 手动/自动 | 谁触发 |
|---------|----------|--------|
| 线索 new → following | **手动** | 招生点"开始跟进" |
| 线索 following → signed | **手动三步** | 招生发合同→线索付款→培训主管确认 |
| 线索 signed → 自动创建学员 | **自动** | 签约成功后自动创建学员记录 |
| 学员 signed → training | **自动** | 报名课程后自动跳转 |
| 学员 training → qualified | **自动** | 所有课程考核通过后自动跳转 |
| 学员 training → failed | **自动** | 考核未通过 |
| 学员 failed → training | **手动** | 重新培训再考 |
| 学员 qualified → converted | **手动** | 招生/讲师/培训主管点"转简历" |
| 学员 signed → converted（跳过培训） | **手动** | 招生/讲师/培训主管点"转简历" |
| 线索任意状态 → lost | **手动** | 招生标记 |

**关键规则：**
- 签约=三步：招生发合同→线索付款→培训主管确认
- 主管未确认=待签约（pending_approval），确认后=已签约（signed）
- 签约成功后自动创建学员记录
- 先签约再培训，学员顺序：signed→training→qualified→converted
- 有"跳过培训直接转简历"分支，课程是加分项不是前置条件
- 一个学员可报多个课程，每个课程独立报名/培训/考核
- 转简历随时可转，不受课程状态限制
- 报名入口：招生代报名+培训主管代报名+学员自助报名
- 课程缴费包含在签约合同中，报名课程不需要单独缴费
- 证书要对应课程合格才能上传，并打上具体证书标签，没通过的课程对应证书不能上传
- 防遗忘机制：学员课程进度看板（学员端+招生端），未报名/未通过高亮提醒；签约超X天未报名自动提醒招生跟进

## 三、经纪人服务线（客户→发单→签约→上户→更换）

| 状态变化 | 手动/自动 | 谁触发 |
|---------|----------|--------|
| 客户 new → following | **手动** | 经纪人点"开始跟进" |
| 客户 following → matching | **自动** | 经纪人发单后自动跳转 |
| 订单 → open | **自动** | 发单后自动创建 |
| matching → interviewing | **手动** | 经纪人安排面试 |
| interviewing → signed | **手动** | 经纪人确认签约+阿姨手机确认 |
| 客户 → converted | **自动** | 签约后自动跳转 |
| 订单 → signed | **自动** | 签约后自动跳转 |
| 阿姨 → working | **自动** | 签约后自动跳转 |
| 中介合同 → active | **自动** | 签约后自动跳转 |
| signed → open（换阿姨） | **手动** | 经纪人点"更换阿姨"后订单回到open |
| 旧签约 → replaced | **自动** | 更换时自动标记，记下户时间 |
| 旧阿姨 → idle | **自动** | 更换时自动跳转 |
| 新签约 → active | **自动** | 新签约后自动标记，记上户时间 |
| 新阿姨 → working | **自动** | 新签约后自动跳转 |
| 其他推荐 → rejected | **自动** | 签约1个后其他自动拒绝 |
| 客户 → completed | **自动** | 合同到期后系统自动标记 |
| 客户 → closed | **手动** | 经纪人标记流失 |
| 订单 → cancelled | **自动** | 客户closed后自动 |

**关键规则：**
- 无电子合同功能，客户签约环节暂隐藏，经纪人确认签约+阿姨手机确认即可
- 签约=上户中，不分开
- 一个订单对应多条签约记录（1对多），当前只有1条active
- 更换阿姨在同一个订单内完成，合同期不变
- 合同期限不固定，按实际合同约定（有start_date和end_date）
- 客户:合同=1:N，合同:订单=1:1，换阿姨时合同同步换人
- 发单后客户状态自动变matching
- 签约后客户状态自动变converted
