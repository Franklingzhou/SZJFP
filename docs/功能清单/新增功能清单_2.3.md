## 一、任务4：简历审核改造（2026-06-23）

### 功能描述
将阿姨简历的创建/修改从直接写入改为审核流程，所有变更需管理员审批后才能生效。

### 数据库变更
| 表名 | 新增字段 | 类型 | 说明 |
|------|----------|------|------|
| resume_reviews | proposed_data | JSONB | 提议的新数据值 |
| resume_reviews | original_data | JSONB | 原始数据值（修改时） |
| resume_reviews | changed_fields | TEXT[] | 变更的字段名数组 |

### API变更
| 接口 | 变更内容 | 状态 |
|------|----------|------|
| POST /api/workers | 不再直接写workers表 → 创建空worker记录 + 提交resume_reviews审核 | ✅ |
| PUT /api/workers | 不再直接update → 提交resume_reviews审核（含diff） | ✅ |
| GET /api/resume-reviews | 返回加proposed_data/original_data/changed_fields | ✅ |
| POST /api/resume-reviews | 支持新字段参数 | ✅ |
| PUT /api/resume-reviews | 审核通过时从proposed_data写入workers表 | ✅ |

### 前端变更
| 页面 | 变更内容 | 状态 |
|------|----------|------|
| 管理员-简历审核 | 展示变更摘要（新旧值对比） | ✅ |
| 招生-简历录入 | 提交后显示"审核中"状态 | ✅ |
| 招生-简历编辑 | 提交后显示"审核中"状态 | ✅ |

## 三、客户表补字段（2026-06-22）

### 功能描述
客户表增加状态流转、来源追踪和经纪人归属字段，支持客户生命周期管理。

### 数据库变更
| 表名 | 新增字段 | 类型 | 说明 |
|------|----------|------|------|
| customers | status | VARCHAR(20) DEFAULT 'new' | 状态：new/matching/signed/lost |
| customers | source | VARCHAR(50) | 来源 |
| customers | agent_id | VARCHAR(36) | 关联经纪人(users.id) |

### API变更
| 接口 | 变更内容 | 状态 |
|------|----------|------|
| GET /api/customers | 加status/source/agent_id筛选 | ✅ |
| POST /api/customers | 默认status='new'，agent_id默认当前用户 | ✅ |
| PUT /api/customers | allowedFields加3字段+status枚举校验 | ✅ |

### 业务逻辑
- 客户创建时status默认'new'，agent_id默认当前登录用户
- 发单后客户status → matching
- 签约后客户status → signed
- 流失客户status → lost

## 六、P0安全漏洞修复（2026-06-20）

### 漏洞修复清单
| 漏洞 | 严重程度 | 修复内容 | 状态 |
|------|----------|----------|------|
| Token无签名校验 | P0 | generateToken改用jose库HS256 JWT签名 | ⬜ |
| 开发模式后门 | P0 | 删除parseLegacyToken/verifyLegacyToken后门函数 | ⚠️ |
| PUT接口无白名单 | P0 | 新增sanitizeUpdateFields白名单机制 | ⬜ |

> ⚠️ HS256 JWT 未落地：当前 `generateToken()` 仍为 Base64 伪签名。`sanitizeUpdateFields` 函数不存在于 `auth-middleware.ts`。`开发模式后门`：`parseLegacyToken`/`verifyLegacyToken` 已不存在，但 `requireAuth()` 第 230 行仍保留 dev 模式兜底返回 admin001。

### API安全加固
| 接口 | 加固内容 | 状态 |
|------|----------|------|
| PUT /api/workers | 只允许更新name/phone/age等业务字段 | ⬜ |
| PUT /api/orders | 只允许更新title/status等业务字段 | ⬜ |
| PUT /api/reviews | 只允许更新评分等业务字段 | ⬜ |

> ⚠️ 上述 3 项依赖 `sanitizeUpdateFields`，该函数尚未实现。
