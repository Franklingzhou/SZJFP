# PC管理后台 — 干了什么、有什么问题

> 2026年7月1日 · 把50个后台页面全翻了一遍

---

## 一、先说结论

后台一共有 **50 个页面**，我挨个打开看了一遍：

- ✅ **36 个页面能用**，功能基本齐全
- 🔧 **6 个页面有小毛病**，不太灵光
- ⚠️ **3 个页面就是一个跳转**，点进去直接跳到别处
- ❌ **5 个页面要么是空壳、要么根本没有**

最大的两个问题：
1. **没有任何人真正测过这些页面**——是的，连点都没点过
2. **有些页面的数据根本存不住**——刷新就没了

---

## 二、每一页干嘛的、有什么毛病

下面按菜单顺序一个一个说。你看得懂的部分看用途，看不懂的跳过就行。

## 三、问题排行榜（从严重到轻微）

### 🔴 不修不行

| 问题 | 大白话说 |
|------|---------|
| **角色权限配置是假的** | 你在页面上勾了一堆权限，一刷新全没了。因为根本没存到数据库，只在浏览器内存里。 |
| **场地管理数据不保存** | 你添加了一个场地、做了一个预约，刷新——全没了。同样的毛病。 |
| **三套客户管理分不清** | 菜单上有三个"客户管理"，你自己都不知道该用哪个，功能还重叠。 |
| **三套合同管理也分不清** | 建个合同不知道该去哪个页面，三个页面干的事情差不多。 |
| **跟进记录写了白写** | 客户页里写的跟进记录，刷新就没了。 |
| **JWT解析取错了字段** | 两页取不到当前用户是谁，部分功能可能废了。 |

### 🟡 不修难受

| 问题 | 大白话说 |
|------|---------|
| **弹窗全是系统自带的** | 成功弹 `alert("成功")`、失败弹 `alert("失败")`、删除弹 `confirm("你确定？")`。浏览器自带的那种白框框，丑且体验差。90%的页面都这样。 |
| **成功/失败判断不统一** | 有的页面判断 `data.ok`，有的判断 `data.success`，有的判断 `result.data`。乱。 |
| **token存的名字不一样** | 有的叫 auth_token，有的叫 miniapp_token，有的叫 token。客户端服务端对不上。 |
| **系统设置页一个文件1400行** | 11个功能模块挤在一个文件里，谁改谁晕。 |
| **复制文本用了过时方法** | `execCommand('copy')` 已经被浏览器淘汰了，应该用新的 clipboard API。 |

### 🟢 可以缓缓

| 问题 | 大白话说 |
|------|---------|
| 证书审核时过滤了列表再点通过，结果通过的可能是另外一个人 |
| 佣金费率和诚信分规则硬编码在前端，如果后端改了就不一致了 |
| 操作日志搜索有翻页bug |
| 培训合同新建用了PUT（不规范，应该用POST） |
| 审核拒绝理由用prompt弹窗输入（没法换行，体验差） |
| 有些地方操作完直接 `location.reload()` 整页刷新 |

## 五、接下来怎么修（建议顺序）

### 第一优先级：修"假的"功能
1. **角色权限** → 让它真正存到数据库
2. **场地管理** → 同上
3. **客户页跟进记录** → 同上

### 第二优先级：合并重复页面
4. 三套客户管里面选一个保留，另外两个删掉或重定向
5. 三套合同管里理清边界，别互相打架

### 第三优先级：代码规范化
6. 把100多个 `alert()` 换成正常的弹窗组件
7. 统一token存储和API调用方式
8. 把1400行的系统设置页拆成多个文件

---

*查了50个页面，发现24个问题。没有一个是API的问题——API大部分都正常。问题全在前端页面上，而且没人测过。*
