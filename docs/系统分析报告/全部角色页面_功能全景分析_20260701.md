# 全部角色页面 — 功能全景分析（大白话版）

> 2026年7月1日 · 把8个角色共100+个页面全翻了一遍

---

## 一、总体情况

整个平台分 **8 个角色**，每个角色有自己的手机端页面：

| 角色 | 页面数 | 整体评价 |
|------|:---:|---------|
| 👩 阿姨 | 12 | 简历页大工程，v059修复征信假结果，v060上传已接真实API（视频除外） |
| 💼 经纪人 | 7 | 核心的订单管理接了API，但首页和客户管理全用假数据 |
| 📢 招生代理 | 7 | 跟进了半天，刷新全没——跟进记录不存库 |
| 🎓 讲师 | 8 | 所有按钮都接不了API，全用假数据 |
| 👤 客户 | 5 | 评价提交了但对象是谁都不知道（target_user_id为空） |
| 🏢 培训主管 | 10 | 线索管理有个致命bug：分配后数据回退到初始值 |
| 🔧 阿姨运营 | 5 | 所有菜单链接全是死链（路径拼错了） |
| 🛡️ 管理员 | 50 | [详见另一份报告](./PC端管理后台_页面功能全景分析_20260701.md) |

**最严重的问题：七个角色里，四个角色的页面几乎完全靠假数据撑着。**

## 二、逐角色分析

---

### 👩 阿姨端（12个页面）

阿姨端是页面最多的一个端，也是"看起来最完整"的端——但简历页的问题一箩筐。

#### 首页
- **干什么的**：个人信息卡片 + 快速统计（诚信分/积分/完成单数/好评数）+ 接单大厅 + 我的课程
- **问题**：全部假数据，没调任何API。接单成功、状态切换都是假的，刷新就回到初始状态。

#### 简历页（最大最复杂的页面）
- **干什么的**：填个人资料、传头像/视频/身份证/照片、写工作经验、查征信、提交审核
- **问题**：
  - 🔧 头像/身份证/照片已接真实API（v060），选中文件后通过 `/api/upload` 上传到 Supabase Storage
  - 🔴 视频上传仍不支持（upload API 不认 video MIME type）
  - ✅ 征信查询失败时直接提示"暂不可用"，不再给假结果（v059）
  - 🔴 工作经验是写死在代码里的默认文字，不从数据库加载
  - 🔴 "申请再培训"的请求没带登录信息，后端不知道谁在申请
  - 🟡 提交成功弹 `alert()` 提示，移动端体验极差

#### 接单大厅
- **干什么的**：两Tab——"接单大厅"看订单接单，"我的订单"看自己接的单
- **问题**：
  - 🔴 投递成功弹原生 `alert()`，失败也弹 `alert()`
  - 🔴 "我的订单"没按当前阿姨ID过滤，可能看到别人的订单
  - 🟡 工种筛选选了之后接单按钮消失的交互逻辑有问题

#### 我的订单（推荐记录）
- **干什么的**：看自己被推荐的记录，接受或拒绝
- **问题**：这个页面标题叫"推荐记录"但放的是阿姨视角，阿姨不应该有"接受/拒绝"按钮——这应该是经纪人端的操作。

#### 上户确认
- **干什么的**：确认已接单的阿姨已经开始上户
- **问题**：
  - 🔴 没按当前阿姨ID过滤订单，可能看到别人的
  - 🔴 确认成功弹 `alert('上户确认成功！')`

#### 我的课程
- **干什么的**：看自己报名的课程和成绩
- **问题**：
  - 🔴 取用户ID时用的 localStorage key 是 `miniapp_user_id`，但系统其他地方存的是 `miniapp_userid`（差一条下划线），可能导致永远取不到ID、永远看不到课程

#### 我的合同
- **干什么的**：看自己签署的合同
- **问题**：加载失败时静默失败，用户看到"暂无合同"但其实是接口挂了

#### 我的客户
- **干什么的**：看分配给自己的客户
- **问题**：catch块完全为空 `catch { setCustomers([]) }`，错误被吞了

#### 想培训
- **干什么的**：浏览课程并报名
- **问题**：
  - 🔴 报名时只传了课程ID，没传自己是谁——后端不知道谁报了名
  - 🔴 报名成功后重新加载课程列表（毫无意义，应该刷新报名状态）

#### 个人中心
- **干什么的**：看个人信息、跳转各功能、看评价、退出登录
- **问题**：
  - 🔴 全部假数据
  - 🔴 评价隐藏功能调的是假数据操作，刷新后隐藏状态丢失
  - 🟡 状态显示用了 `idle/working`，首页用了 `available/busy/paused`——两处不一致

### 💼 经纪人端（7个页面）

经纪人端的核心业务页面（订单管理）接了真实API，但首页和客户管理完全靠假数据。

#### 首页
- **干什么的**：概览卡片 + 一键发单 + 修改订单 + 录客户
- **问题**：
  - 🔴 全部假数据，没调API
  - 🔴 修改订单弹窗是假的——所有输入框用 `defaultValue` 而非受控组件，点"保存"只弹 `alert('修改成功')`，实际什么都没存

#### 客户管理
- **干什么的**：看客户列表、编辑客户、写跟进记录、一键发单
- **问题**：
  - 🔴 全部假数据
  - 🔴 编辑客户直接改的 mock 数组，刷新全丢
  - 🔴 跟进记录只存在浏览器内存里，页面一切换就没了
  - 🔴 "一键发单"是假的——只显示1.5秒成功动画，没创建任何订单

#### 合单大厅
- **干什么的**：看订单、推荐阿姨给客户
- **问题**：共享组件 HallPage，硬编码了 `currentUserId="a001"`，不管实际登录的是谁

#### 订单管理
- **干什么的**：看订单列表、发单、推荐阿姨、签约
- **这是经纪人端唯一接了真实API的核心页面**，但有一些小问题：
  - 🟡 创建订单的工种字段是自由输入框，不是下拉选择，可能输入不合规的值
  - 🟡 推荐阿姨时按 `status === 'active' || 'available'` 过滤，但阿姨的实际状态值是 `idle/working/paused`，可能一个阿姨都搜不到
  - 🟡 错误处理不完善，catch 里只设空数组

#### 阿姨管理
- **干什么的**：看自己的阿姨和全部阿姨
- **问题**：全部假数据，连搜索和筛选都没有

#### 个人中心
- **干什么的**：看个人信息、评价、退出登录
- **问题**：
  - 🔴 评价和佣金统计全用假数据
  - 🟡 诚信分调了真实API，但失败时静默回退到1000

### 📢 招生代理端（7个页面）

招生代理端比经纪人端更严重——核心功能"跟进记录"完全不存库。

#### 首页
- **干什么的**：概览卡片 + 录线索 + 线索列表 + 培训订单 + 佣金
- **问题**：
  - 🔴 大量 `alert()` 和 `confirm()` 原生弹窗（10+处）
  - 🔴 数据混用：写入走API，展示看mock——两边不同步
  - 🔴 签约后的数据刷新不靠谱，UI可能还是旧状态
  - 🟡 头部3个统计数字点了没反应

#### 跟进管理（招生端最重要的页面）
- **干什么的**：线索全生命周期——搜线索、写跟进、改状态、签合同
- **问题**：
  - 🔴 **跟进记录写完了，刷新就没了！**——只存本地 state，没调任何API保存。这是招生端的核心业务，这个bug等于废了半个系统
  - 🔴 线索数据回退：新增/修改线索后直接 `setLeads([...mockRecruiterLeads])`，之前所有的修改全部被初始假数据覆盖
  - 🔴 签约后同样的问题——数据回退
  - 🔴 "我的发单"编辑保存没调API
  - 🟡 身份证识别只在前端算了校验码，没调OCR实名API

#### 合单大厅
- **干什么的**：跟经纪人端共用 HallPage
- **问题**：硬编码 `currentUserId="r001"`

#### 培训推荐
- **干什么的**：看课程列表，推荐阿姨去培训
- **问题**：
  - 🔴 全部假数据
  - 🔴 "推荐阿姨"和"详情"两个按钮点了**没有任何反应**——全是死按钮

#### 学员管理
- **干什么的**：看自己招的学员和全部阿姨
- **问题**：
  - 🔴 全部假数据
  - 🔴 学员卡片跳转到简历用的是线索ID而不是阿姨ID，点进去会404
  - 🟡 筛选面板的状态选项和实际数据对不上

#### 个人中心
- **干什么的**：个人信息、统计、评价
- **问题**：全部假数据，诚信分调了API但失败时静默

### 👤 客户端（5个页面）

客户端用户量最大，但问题也很严重——评价写了不知道写给谁。

#### 首页
- **干什么的**：问候语 + 经纪人卡片 + 订单列表 + 待评价列表 + 服务中的阿姨
- **问题**：
  - 🔴 全部假数据，不调任何API
  - 🔴 经纪人永远是 `mockAgents[0]`（第一个），不会按当前客户匹配
  - 🔴 订单硬编码 fallback 到 `customerId === 'c001'`，任何人登录都看c001的订单
  - 🟡 状态映射缺失 `signed` 和 `interviewing`，某些订单状态会直接显示英文

#### 我的订单（客户端最重要的页面）
- **干什么的**：看订单、看签约记录、评价阿姨
- **问题**：
  - 🔴 提交评价时 `target_user_id` 写的是空字符串 `''`！注释说"后续从订单取 worker_id"，但一直没实现。这意味着客户写的评价不知道写给谁的
  - 🔴 评价失败后弹窗照样关闭，用户以为评价成功但实际没成
  - 🟡 订单状态 `in_service`/`in_progress` 永远不会从API返回，只有假数据里才有

#### 我的合同
- **干什么的**：看自己签的合同
- **问题**：
  - 🔴 catch 块完全为空 `catch { setContracts([]) }`，接口挂了就显示空列表

#### 个人中心
- **干什么的**：个人信息 + 统计 + 退出登录
- **问题**：
  - 🔴 订单数和已评价数都用假数据
  - 🔴 已评价数统计用了 `targetId` 过滤，应该是 `reviewerId`——所以已评价数永远为0

#### 评价中心
- **干什么的**：看别人给自己的评价
- **问题**：没什么大毛病，但加载失败时静默

### 🏢 培训主管端（10个页面）

培训主管的线索管理是核心页面，但有个致命bug——操作后数据回退。

#### 首页
- **干什么的**：概览 + 审批入口 + 招生概况 + 线索列表
- **问题**：全部假数据，待审批数字硬编码为2

#### 线索管理（培训主管端最重要的页面）
- **干什么的**：管理所有招生线索——搜索、分配招生人员、签约、写跟进、发合同

  - 🔴 写跟进记录后同样的问题——数据回退
  - 🔴 签约后同样的问题——数据回退

- **干什么的**：确认合同到账/驳回

- **干什么的**：审批课程通过/驳回
- **问题**：

- **干什么的**：审批排课
- **问题**：
  - 🟡 判断成功用了 `result.ok`（这是 Response 对象的属性，不是 JSON 的），虽然碰巧能工作但逻辑不对

#### 教学管理
- **干什么的**：看课程和学员
- **问题**：全部假数据，课程-学员匹配逻辑基本无效

#### 学员管理
- **干什么的**：搜学员、转简历
- **这是培训主管端唯一接了真实API的页面**，基本能工作

#### 个人中心
- **干什么的**：个人信息
- **问题**：待审合同数用假数据

## 三、全平台共性问题排行

### 🔴 要命的（含第59轮修复状态）

| 问题 | 影响范围 | 大白话 | 状态 |
|------|---------|--------|:---:|
| **大量页面用假数据** | 阿姨、经纪人、招生、讲师、客户、阿姨运营——共40+页面 | 你在页面上看到的数字全是写死的，跟数据库没有任何关系 | ⬜ 部分修复 |
| **跟进记录不存库** | 招生代理、经纪人 | 业务人员写了半天跟进，一刷新全没了 | ✅ |
| **上传功能全是假的** | 阿姨简历页 | 头像、身份证、照片、视频——选了文件但实际上没传到服务器 | ✅ v060已修复(视频除外) |
| **征信查询失败给假结果** | 阿姨简历页 | 接口挂了，页面给你看一段硬编码的征信报告，让你以为查成功了 | ✅ |
| **培训主管线索管理数据回退** | 培训主管 | 做任何操作都会把数据重置成初始值，等于这个页面完全不可用 | ✅ |
| **阿姨运营端全是死链** | 阿姨运营 | 菜单上所有链接都是404，这个角色根本用不了 | ✅ |
| **客户评价不知道写给谁** | 客户端 | target_user_id 是空的，评价保存了但关联不到任何人 | ✅ |

### 🟡 难受的

| 问题 | 影响范围 | 大白话 |
|------|---------|--------|
| **alert() 弹窗满天飞** | 全平台100+处 | 成功弹alert、失败弹alert、确认弹confirm——浏览器自带白框，移动端体验极差 |
| **空catch静默失败** | 全平台60+处 | 接口挂了，页面什么都不说，用户以为一切正常 |
| **硬编码用户ID** | hall页面（4个角色共用） | currentUserId 写死了，不认当前登录用户 |
| **localStorage key不统一** | 阿姨端 | 有的存 `miniapp_userid`，有的读 `miniapp_user_id`——永远取不到值 |
| **认证方式不统一** | 全平台 | 有的用 `x-session`，有的用 `Authorization: Bearer`，有的什么都不带 |
| **按钮没有功能** | 讲师端、招生端 | 点了按钮没任何反应，onClick都没写 |
| **状态值不一致** | 阿姨端 | 有的用 idle/working，有的用 available/busy——同一个东西叫法不同 |

## 四、每个角色最影响使用的问题（含第59轮修复状态）

| 角色 | 核心问题 | 严重程度 | 第59轮 |
|------|---------|:---:|:---:|
| 阿姨 | 简历上传全假、征信给假结果、课程页取不到ID | 🔴 | 征信✅ key✅ 上传✅(v060) |
