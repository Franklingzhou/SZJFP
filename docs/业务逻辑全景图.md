## 一、核心概念速查

### 1.1 一句话定位

**连接阿姨(家政服务人员)、经纪人、招生代理、培训讲师和客户的多角色协作系统。**
通过佣金分账、诚信体系、积分激励等机制构建家政服务生态闭环。

### 1.2 "双平台、单系统" 架构

| 平台 | 路径 | 角色 | 技术形态 |
|------|------|------|---------|
| **PC 管理后台** | `/admin/` | admin, agent, recruiter, instructor, training_supervisor, worker_operator (6个内部角色) | Next.js App Router, shadcn/ui |
| **移动端 (H5/小程序)** | `/m/` | worker, customer (2个外部角色) | 微信小程序 / 移动H5 |

## 二、8 种角色体系

| 角色标识 | 中文名 | 端侧 | 核心定位 | 关键权限 |
|---------|--------|------|---------|---------|
| `admin` | 平台管理员 | PC | 全平台管理，唯一能审批所有操作的角色 | 全部权限 |
| `agent` | 经纪人 | PC | 核心中间人，**唯一能发单** | orders:write（独有）、customers:read/write |
| `recruiter` | 招生 | PC | 招聘学员和阿姨，线索管理 | leads:read/write（仅自己）、enrollments:write |
| `instructor` | 讲师 | PC | 培训阿姨，管理课程和学员，考核打分 | courses:read/write、enrollments:grade、grading:write |
| `training_supervisor` | 培训主管 | PC | 审批合同/课程，全量查看线索/学员/课程 | courses:assign、contracts:approve、leads:read(全量) |
| `worker_operator` | 阿姨运营 | PC | 管理阿姨档案，推荐阿姨给订单 | workers:read/write、recommendations:write |
| `worker` | 阿姨 | 移动端 | 被服务对象，投递/自荐，管理自己简历 | orders:read（大厅）、enrollments:write（自助报名） |
| `customer` | 客户 | 移动端 | 被服务对象，只看已签约订单 | orders:read（仅自己的）、profile:write |

### 2.1 关键规则
- **所有内部角色均可新建/查看/修改阿姨简历**（6个PC角色共享阿姨库写权限）— 任何新建或修改都触发简历审核，由管理员审批后才生效
- **阿姨接单状态变更也走简历审核** — 暂停/恢复接单视为简历修改，提交后进入 resume_reviews，仅管理员可批
- **管理员和培训主管不参与互评**
- **阿姨运营不能查看线索列表**（线索由招生跟进）
- **仅经纪人能发单**（admin 也不能，orders:write 只授给 agent）
- **阿姨 pending 状态不可投递订单**
- **学员独立于简历体系** — 报名/考核/转班等操作由培训主管审批即可，不走管理员审批
- **评价需管理员审核上线** — 评价发布后默认隐藏(pending)，管理员审核通过后才对外可见；管理员可随时隐藏/取消隐藏评价

### 2.2 角色分类（内部 vs 外部）

| 分类 | 角色 | 说明 |
|------|------|------|
| **内部** | admin, agent, recruiter, instructor, training_supervisor, worker_operator (6个) | 平台运营方，协助管理阿姨、运营平台 |
| **外部** | worker, customer (2个) | 平台使用者，阿姨维护自己简历，客户查看分享的简历 |

### 2.3 证书上传权限（简历范畴）

证书属于阿姨简历的一部分，存储为 `workers.certificates` JSONB字段：

| 操作 | 内部6角色 | 阿姨 | 客户 | 管理员 |
|------|:---:|:---:|:---:|:---:|
| 上传/修改/删除 | ✅ | ✅ | ❌ | ✅ |
| 审核通过 | — | — | — | ✅ |

- 任何人上传证书后状态为 `pending`，走**简历审核流程**
- 管理员审核 → `approved` / `rejected`
- 讲师考核**不发证**，仅打分（excellent/qualified/failed），考核结果记入培训记录
- 证书与培训记录分开展示在简历页

## 三、核心状态机（7个）

### 3.1 阿姨状态 (WorkerStatus)

```
pending(待审核) ──审核通过──▶ available(空闲可用)
                                │
                                ├──────────▶ working(工作中)
                                │
                                ├──────────▶ paused(暂停接单)
                                │              (阿姨申请→管理员审核)
                                │
                                └──────────▶ blacklisted(黑名单)
                                               (诚信分低于阈值/管理员手动拉黑)
```

> 共 5 个状态。`available`：空闲可投递；`paused`：阿姨（或任意内部角色）申请暂停接单 → 自动生成简历审核记录 → 管理员在 resume_reviews 中审批 → 通过后状态变更为 paused。恢复接单同理，也走 resume_reviews 审核。

### 3.2 订单状态 (OrderStatus)

```
created(已创建) → open(已发布大厅) → interviewing(面试/匹配中)
                                          │
                                       签约确认 ──────▶ signed(已签约)
                                          │
             ┌────────────────────────────┼────────────────────────────┐
             ▼                            ▼                            ▼
      in_progress(进行中)          completed(已完成)            cancelled(已取消)
```

> `created`：经纪人刚创建，尚未发布到大厅。`interviewing`：已指定阿姨，待确认签约。

### 3.3 线索状态 (LeadStatus)

```
new(新线索) → following(跟进中) → signed(已签约,学员诞生) → converted(已转化,转成简历进大厅)
                │                    │
                └── lost(已流失) ◀───┘
```

### 3.4 客户状态 (CustomerStatus)

```
new(新客户) → following(跟进中) → matching(匹配中/已发单) → converted(已签约)
                                                             │
                                                             ▼
                                                        completed(已完成)
                                                             │
                                         closed(已关闭) ◀───┘
```

### 3.5 报名/学员状态 (EnrollmentStatus)

```
enrolled(已报名) → attending(学习中) → excellent(优秀) → 转简历
                                    → qualified(合格) → 转简历
                                    → failed(未通过) → 可重学
                                    → dropped(已退学)
```

### 3.6 推荐状态 (RecommendationStatus)

```
pending(待审核) → accepted(已通过) / rejected(已拒绝) / signed(已签约)
```

> `accepted`：管理员审核通过或经纪人自推荐自动确认（详见 §7.3）。

### 3.7 简历审核状态 (ResumeReviewStatus)

```
draft(草稿) → pending(待审核) → approved(已通过) / rejected(已驳回)
```

## 四、4条核心业务流程

### 流程1：招生培训线（线索 → 签约 → 培训 → 转简历）

```
招生录入线索(姓名+手机号)
  → 跟进线索(添加跟进记录)
  → 签约：发培训合同 → 主管代确认（学员只需查看，不需操作）
  → 报名课程(招生/主管/学员自助)
  → 讲师考核打分（excellent优秀 / qualified合格 / failed不合格，不合格可重学）
  → 考核结果记入培训记录（非证书）
  → [防遗忘] 签约后超期未报名→自动提醒

转简历（随时可触发）：
  ● 签约后直接转（跳过培训）
  ● 培训中/通过后转
  ● 未签约线索也可直接新建简历（不走管线）
  ↓
converted(已转化) → 自动创建 worker(pending) → 管理员审核 → worker=available(进大厅)
```

### 流程2：经纪人服务线（客户 → 发单 → 推荐 → 签约 → 上户）

```
经纪人录入客户
  → 发单(创建订单，状态=open)
  → 各角色推荐阿姨给订单
  → 签约：经纪人确认 + 阿姨手机确认
→ 生成中介合同
→ [可选] 更换阿姨（同一订单内完成）
  → [可选] 客户流失 → 级联关闭
  → 合同到期处理
  → 退款（培训费/中介费/保证金）
```

### 流程3：阿姨端（简历 → 投递 → 上户）

```
查看/编辑简历(修改触发审核)
  → 订单大厅投递(available状态)
  → 订单大厅自荐
  → 想培训(自动创建或回退线索)
  → pending不可投递
  → 查看课程/自助报名
→ 查看合同(培训+中介双Tab)
→ 签约确认
```

### 流程4：平台管理线

```
简历审核(含diff对比)
  → 合同审核(培训合同)
  → 课程审核
  → 角色注册审核
  → 评价审核(通过上线/拒绝打回/隐藏敏感内容)
  → 退款审核(pending→approved/rejected→completed)
  → 暂停接单审核
  → 页面权限配置
  → 通知管理
  → 数据导出
  → 佣金配置
  → 平台收费确认
```

## 六、关键业务规则速查

| # | 规则 | 说明 |
|---|------|------|
| 1 | 发单权限 | **仅经纪人能发单**，admin 也不能 |
| 2 | 订单大厅 | 统一一个大厅，所有角色共享同一订单池 |
| 3 | 简历审核 | 所有角色的新建/修改都需管理员审核 |
| 4 | 签约=上户 | 签约即上户中，不分开 |
| 5 | 线索唯一性 | 一个手机号只能一条线索（防撞单） |
| 6 | 客户不唯一 | 同一客户可被多个经纪人创建 |
| 7 | 更换阿姨 | 同一订单内完成，合同期不变 |
| 8 | 推荐签约联动 | 签约1人后，同订单其他推荐自动拒绝 |
| 9 | 订单关闭联动 | 订单关闭时，所有关联推荐自动 rejected |
| 10 | 预注册机制 | 手机号先录入，用户登录即认领 |
| 11 | 课程缴费 | 包含在签约合同中，报名不需单独缴费 |
| 12 | 证书规则 | 证书属于简历范畴，7个角色（6内部+阿姨）均可上传，走简历审核流程。平台不自发证，讲师考核仅打分（excellent/qualified/failed），考核结果体现在培训记录中 |
| 13 | 签约必须关联阿姨 | signed 状态必须设 sign_worker_id |
| 14 | 拒绝推荐必填理由 | rejection_reason 必填 |
| 15 | 平台双20% | 中介费20% + 服务费20%，金额由管理员手动输入确认 |
| 16 | 评价审核上线 | 评价发布后默认隐藏(pending)，管理员审核通过才对外可见；管理员可随时隐藏/取消隐藏某条评价 |

## 七、佣金分账体系

### 7.1 三级分账
```
经纪人(30-50%) + 招生代理(10-20%) + 平台(剩余)
```
按单分账，支持自定义比例。

### 7.2 三方分账（阿姨简历维度）
```
创建人 + 维护人（最后修改人） + 推荐人
```
三人可以是同一人，比例由管理员配置。手动结算：pending / settled。

### 7.3 推荐自动确认规则
- **经纪人自推荐**：经纪人推荐自己的阿姨 → 自动确认通过，跳过双向确认流程

### 7.4 平台收费（双20%）
| 费用类型 | 计算基数 | 收费对象 | 触发时机 |
|---------|---------|---------|---------|
| 中介费分成 | 经纪人签约中介费 × 20% | 经纪人 → 平台 | 签约后 |
| 服务费分成 | 阿姨上户服务费 × 20% | 阿姨 → 平台 | 上户后 |

### 9.1 评价生命周期

```

  → 管理员审核
       ├── 通过 → status=approved, hidden=false（上线对外可见）

```

## 十、诚信分与保证金

- **诚信分**：初始 1000 分，违约扣分，良好表现加分
- **黑名单**：诚信分低于阈值自动拉黑，有冷冻天数
- **保证金**：按角色等级缴纳，可退还
- **积分**：行为获取（完成订单、好评等），可兑换权益
