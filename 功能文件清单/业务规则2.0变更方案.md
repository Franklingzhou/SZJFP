## 一、签约→简历流程变更（P0）

### 1.1 现状

```
线索签约(sign) → 只改leads.status='signed'，不创建任何东西
线索转简历(convert) → 手动触发 → workers POST → worker_audits(pending) → admin审核 → 创建worker记录
外部注册 → worker_audits(pending) → admin审核 → 创建worker记录
```

### 1.2 目标流程

```
线索签约(sign) → 自动创建worker(status=pending) → admin审核 → worker.status=available
外部注册 → worker_audits(pending) → admin审核 → 创建worker记录（不变）
```

- 取消手动"转简历"步骤，签约即自动创建worker
- pending worker在订单大厅可见但不可投递
- admin审核通过后worker.status变为available，可投递

### 1.3 数据库变更

  refund_type VARCHAR(50) NOT NULL,           -- training_fee | agency_fee | deposit

## 五、预注册（验证码自动认领）（P2）

### 5.1 业务规则

- 录入人（招生/经纪人/运营）录入线索/客户/阿姨时只填手机号，不创建auth账号
- 客户/阿姨首次登录用被录入的手机号收验证码
- 验证通过后自动认领对应profile（绑定user_id）

### 5.2 API变更

| API | 变更 |
|-----|------|
| `auth/send-otp` | 已存在，检查手机号是否匹配现有worker/client记录 |
| `auth/verify-otp` | 验证通过后，检查该手机号是否有未认领的worker/client记录，有则自动绑定user_id |
| `auth/register` | 保留（管理员创建账号场景），但C端用户不再走register |

### 5.3 认领逻辑

```
用户输入手机号 → 收验证码 → 验证通过
  → 查workers表 phone=xxx AND user_id IS NULL → 有待认领worker
  → 查clients表 phone=xxx AND user_id IS NULL → 有待认领client
  → 如果只有一种 → 自动绑定
  → 如果两种都有 → 让用户选择身份（阿姨端 or 客户端）
  → 查profiles表 phone=xxx → 有则确认登录
  → 都没有 → 创建新profile（普通用户）
```

### 5.4 前端变更

- C端登录页只保留"手机号+验证码"方式，不显示密码登录
- 后台录入页面去掉"创建账号"相关步骤
