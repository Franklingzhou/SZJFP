## 类别 2：定时任务

项目有 **8 个** Cron Job API（Vercel Cron 触发），全部在 `src/app/api/cron/` 下：

| # | 文件路径 | 建议频率 | 影响的数据库表 | 推测用途 |
|---|---------|---------|--------------|---------|
| 1 | `cron/contract-expiry/route.ts` | 每天凌晨3点 | contracts | 合同到期检查：7天内紧急提醒 + 30天内常规提醒 + 自动标记过期合同 |
| 2 | `cron/contract-unsigned/route.ts` | 每天上午10点 | contracts | 合同超时未确认提醒（默认72小时），培训合同提醒培训主管，中介合同提醒经纪人 |
| 3 | `cron/enrollment-reminder/route.ts` | 每天上午10点 | enrollments, leads | 防遗忘机制：签约超7天未报名阿姨提醒 + 线索超14天未转化提醒 |
| 4 | `cron/enrollment-unscheduled/route.ts` | 每天上午10点 | enrollments, course_schedules | 报名后超7天未排课提醒，通知招生代理 |
| 5 | `cron/fee-overdue/route.ts` | 每天上午9点 | platform_fees | 费用逾期检查：自动标记逾期平台费 + 催缴通知；3天内即将逾期预警 |
| 6 | `cron/lead-unfollowed/route.ts` | 每6小时 | leads | 线索超24小时未跟进提醒，通知对应招生代理 |
| 7 | `cron/order-unmatched/route.ts` | 每6小时 | orders | 订单超48小时未派单提醒，通知经纪人 |
| 8 | `cron/worker-inactive/route.ts` | 每天一次 | workers | 阿姨30天未活跃提醒，通知创建者/经纪人 |

**风险等级：低** — 全部有业务注释，覆盖面合理。但需确认 Vercel Cron 是否已在 `vercel.json` 中配置。

## 类别 7：全局过滤器/拦截器

### 7.1 认证中间件

| 属性 | 值 |
|------|-----|
| **文件路径** | `src/lib/auth-middleware.ts` |
| **作用范围** | 所有 API route handler（88+ 个文件引用） |
| **功能** | JWT 登录态校验 + 角色权限矩阵（50+ 权限 key × 8 种角色） |
| **JSDoc** | ✅ 完整（文件头有详细用法说明，导出函数各有注释） |
| **测试覆盖** | ✅ 有（`tests/api-test/` 下权限测试套件） |
| **导出函数** | `requireAuth`, `requireRole`, `checkPermission`, `requirePermission`, `checkPermissionDetailed`, `unauthorizedResponse`, `forbiddenResponse` |

### 7.2 缺失的全局中间件

项目没有 Next.js 原生 `middleware.ts`（全局中间件）。所有认证/授权逻辑在 handler 内部手动调用 `requirePermission()`。这意味着：
- 静态页面/资源无全局拦截
- 跨域（CORS）未在代码层集中配置（依赖 CloudBase 网关）

**风险等级：低** — 当前架构可接受，handler 级权限检查已覆盖所有 API。
