## 🚨 前置审查：权限校验完整性审计

> 对 `auth-middleware.ts` 中 71 个权限 key + 所有 API 路由做了逐接口审计，发现以下覆盖缺口。

### 中间件两种模式

| 模式 | 函数 | 行为 |
|------|------|------|
| **模式A（手动区分）** | `checkPermissionDetailed()` + 路由内手动判断 | 路由自行返回 401/403，**已全部迁移完**，不再使用 |
| **模式B（自动区分）** | `requirePermission()` | 内部自动区分：未登录→401, 已登录但无权限→403 |

### ⚠️ dev 模式特殊行为（影响测试可靠性）

```
dev 模式（无 SMS_PROVIDER 环境变量）:
  - 无 token 的请求 → requireAuth() 返回 null        → 401 ✅
  - 携带 "dev-admin" token → 直接返回 admin session   → 跳过鉴权 ⚠️
  - 携带其他无效 token → 返回 {role:'guest'} session  → 走 403 而非 401 ⚠️
```

**结论**：dev 模式下的 401 测试必须用 `createClient()`（完全不传 token），不能用 `createClient('fake-token')`。当前测试脚本已按此规范编写，无问题。

## 第二部分：自动化测试边界条件规范

> 确保自动化测试不仅覆盖"正常流程"，也覆盖劣化场景。

### 每个测试套件必须覆盖的边界条件清单

| 边界类型 | 覆盖要求 | 示例用例 |
|----------|---------|---------|
| **输入长度** | 超长字符串(>1000 chars)、空字符串、单字符 | 搜索关键词1001字符截断验证 |
| **数值边界** | 负数、0、MAX_SAFE_INTEGER、浮点精度 | pageSize=-1 应返回400；金额0.001精度 |
| **特殊字符** | SQL注入(`'; DROP TABLE--`)、XSS(`<script>alert(1)</script>`)、Unicode(emoji/零宽字符) | 用户名含emoji应正常保存或拒绝 |
| **类型错误** | 字符串字段传数字、数字字段传字符串、必填字段传null | phone字段传数字13800138000 |
| **并发冲突** | 同一资源同时2次POST、同时PUT+DELETE | 同一订单并发2次 settle |
| **资源不存在** | GET/PUT/DELETE 不存在的ID | GET /api/workers/nonexistent-id → 404 |
| **分页越界** | page=999999, pageSize=0/负数 | 超大页码应返回空列表 |
| **日期越界** | 公元1000年、公元9999年、2月30日 | schedule日期 2026-02-30 → 400 |
| **JSON格式** | 非JSON body、畸形的JSON、嵌套深度>10层 | Content-Type: application/json + body=plaintext |
| **重复提交** | 短时间内相同请求2次（幂等性） | 注册相同手机号连续2次 → 第一次201、第二次409 |

### 权限边界特殊场景

| 场景 | 测试方法 | 预期 |
|------|---------|------|
| 无 Authorization header | `createClient()` 不带 token | 401 |
| 空 Authorization header | `createClient('')` | 401 |
| 伪造 JWT token | `createClient('fake.jwt.token')` | 401 (prod) / 403 (dev guest) |
| 过期 token | 手动构造已过期 JWT | 401 |
| 跨角色越权 | agent token 访问 admin-only 接口 | 403 |
| `orders:write` 仅 agent | admin token POST orders | ⚠️ 当前矩阵 admin 无此权限 → 403 |

> ⚠️ **发现**：`orders:write` 仅允许 `agent` 角色，连 `admin` 也不允许。需确认这是否符合业务预期（管理员是否应该能发单？），如果不符合需要修正 `ROLE_PERMISSIONS` 并更新测试。
