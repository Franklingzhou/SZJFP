# 端到端黄金路径 — 上线前手动验证

> **版本**：v1.0 (2026-06-30 v049)
> **目的**：每次上线前，有一个人按此文档操作一遍，覆盖平台最核心的业务链路
> **预计耗时**：25-35 分钟（前提：已运行 `node scripts/seed_test_scenarios.cjs` 创建测试数据）
> **先决条件**：部署已完成，数据库可读写

---

## 路径1：核心业务流程（经纪人→签约→完成）🔴 最重要

### Step 1: 经纪人登录
| 操作 | 期望 |
|------|------|
| 打开 `/m/login`，输入 `13600001234` / `888888` | 跳转 `/m/agent`，显示工作台 |

### Step 2: 查看订单大厅
| 操作 | 期望 |
|------|------|
| 进入「合单大厅」或 `/m/agent/hall` | 能看到 open 状态订单（种子脚本创建了 `seed_v2_order_open`） |

### Step 3: 创建订单
| 操作 | 期望 |
|------|------|
| 点「发单/创建合单」→ 填写信息 → 提交 | 订单创建成功，出现在列表 |

### Step 4: 阿姨投递
| 操作（切换到阿姨账号 `13800005678`）| 期望 |
|------|------|
| 进入 `/m/worker` → 接单大厅 | 能看到 Step 3 创建的订单 |
| 点「投递」| 投递成功提示 |

### Step 5: 经纪人发起签约
| 操作（切回经纪人 `13600001234`）| 期望 |
|------|------|
| 订单详情 → 选择投递的阿姨 → 「发起签约」 | 订单状态 → interviewing |
| 「生成合同」→ 填写合同信息 → 保存 | 合同创建，待确认 |

### Step 6: 确认签约
| 操作 | 期望 |
|------|------|
| 经纪人确认 → 阿姨确认（阿姨端确认入口） | 订单 → signed，合同 → active |

### ✅ 路径1通过标准
- [ ] 订单状态正确流转：`open → interviewing → signed`
- [ ] 合同状态正确：`draft → pending_approval → active`
- [ ] 签约后阿姨状态 → working

## 路径2：培训流程（讲师→考核→转简历）🔴 v049重点

### Step 7: 讲师查看学员
| 操作（`13700007890`）| 期望 |
|------|------|
| 进入 `/m/instructor/students` | 能看到学员列表（种子脚本创建了 `seed_v2_enroll_s1`） |
| 查看学员详情 | 显示已通过的课程和成绩 |

### Step 8: 转简历（v049核心）
| 操作 | 期望 |
|------|------|
| 合格学员 → 点「转简历」按钮 | 弹出确认对话框 |
| 确认 | 成功提示，学员→converted，worker.status=available |

### Step 9: 培训主管转简历（v049）
| 操作（`13100001111`）| 期望 |
|------|------|
| 进入 `/m/training_supervisor/students` | 能看到所有学员 |
| 合格学员 → 点「转简历」| 同样能调用 API，成功转换 |

### Step 10: 课表审核（v049新增页面）
| 操作（`13100001111`）| 期望 |
|------|------|
| 进入 `/m/training_supervisor/approval/course-schedules` | 页面不白屏，能看到待审核课表 |
| 点「通过」/「驳回」| 课表状态更新 |

### ✅ 路径2通过标准
- [ ] 讲师端学员列表有「转简历」按钮，可成功调用 API
- [ ] 培训主管学员列表有「转简历」按钮
- [ ] 课表审核页 `/m/training_supervisor/approval/course-schedules` 可用
- [ ] 转简历后 worker.status=available
